# Repository Rules

But: short, actionable rules to be enforced by humans and AI tools.

## 1. Project Scope & Stack

- **Languages**: Plain JavaScript only. **NO TypeScript** (`*.ts`, `*.tsx`).
- **Modules**: ES Modules (ESM) exclusively. No `require()` or `module.exports`.
- **Infrastructure**: Netlify (Functions/Hosting), Supabase (DB/Auth/RLS).
- **Environment**: Node.js 18+.

## 2. Coding Standards

- **Imports**: Use top-level `import` / `export`.
- **Extensions**: Keep files as `.js`. No `.mjs` or `.jsx` required if `package.json` has `"type": "module"`.
- **Formatting**: Prettier is used. Run `npm run format` before committing.
- **Linting**: ESLint for JS, Stylelint for CSS. Run `npm run lint`.
- **Git**: Husky hooks run `lint-staged` on commit.

## 3. Architecture Principles

- **Monorepo**: Multi-package structure (`apps/`, `packages/`).
- **AI Integration**: AI logic is primarily in `packages/room/lib/ai/`.
- **State Management**: Prefer local state or Supabase Realtime/RLS over complex global stores.
